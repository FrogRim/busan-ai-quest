Build Plan: Busan Quest Web Game



Build this as a QR-first mobile web app/PWA. Users scan a QR code at the Codex event, open the game in browser, allow location, then receive a short Busan walking quest generated by AI. The game should feel like an exploration game, not a normal travel guide.



Core Game Loop



User scans QR code.

App opens mobile web game.

User logs in with nickname, email, or event code.

App asks for location permission.

Agent 1 generates a local mission set based on nearby area.

User sees a map with 3 mystery mission slots.

User walks around Busan and explores.

Agent 2 acts as a voice companion while walking.

User submits photo + answer for each mission.

Agent 3 judges whether the submission is valid.

User completes all 3 missions and gets score, badge, share screen, or event reward.

Example for Jagalchi Market:



Find a local craft or souvenir shop.

Find a food place connected to seafood or Busan street food.

Find a tourist/photo spot nearby.

User must take a picture and answer a short question for each.

Three Agent Design



Agent 1: Mission Designer



Purpose: Create missions based on location.



Inputs:



User GPS location

Nearby district name

Known Busan POIs

Event rules

Difficulty level

Time limit

Walking radius

Output should be structured JSON:



{

&#x20; "area": "Jagalchi Market",

&#x20; "theme": "Sea Market Explorer",

&#x20; "missions": \[

&#x20;   {

&#x20;     "id": "craft\_shop",

&#x20;     "title": "Find a local craft or souvenir shop",

&#x20;     "clue": "Look for something handmade or locally themed.",

&#x20;     "allowed\_radius\_m": 500,

&#x20;     "proof\_required": \["photo", "short\_answer"]

&#x20;   }

&#x20; ]

}

Important rule: Agent 1 can create clues, but should not reveal exact answers or exact business names unless the game mode allows it.



Agent 2: Voice Companion



Purpose: Make the walk feel alive.



Behavior:



Talks like a friendly local exploration companion.

Shares Busan culture, food, history, and fun facts.

Encourages the player.

Can react to location progress.

Must not directly reveal the 3 mission answers.

Can give soft hints only after the user struggles.

Example guardrail:



Do not name the exact target place unless the player has already found it.

Do not say “go to this shop.” Give atmospheric clues only.

Use Realtime API with WebRTC for browser voice. The companion can also have text mode fallback if the event Wi-Fi or microphone permission fails.



Agent 3: Mission Judge



Purpose: Validate photo + answer.



Inputs:



Mission definition

User GPS at submission time

Photo

User short answer

Time since mission started

Judging output:



{

&#x20; "passed": true,

&#x20; "confidence": 0.87,

&#x20; "reason": "The image appears to show a seafood stall and the answer mentions local grilled fish.",

&#x20; "feedback": "Nice find. This matches the food mission.",

&#x20; "points\_awarded": 100

}

Rules:



Do not require perfect accuracy.

Allow creative valid answers.

Reject obvious cheating, screenshots, unrelated photos, or photos far outside the mission area.

If confidence is low, ask for one extra answer or another photo instead of hard failing.
